# 8.7.2 简化路径

**题目**：给你一个字符串 `path` ，表示指向某一文件或目录的 Unix 风格 **绝对路径** （以 `'/'` 开头），请你将其转化为 **更加简洁的规范路径**。

在 Unix 风格的文件系统中规则如下：

- 一个点 `'.'` 表示当前目录本身。
- 此外，两个点 `'..'` 表示将目录切换到上一级（指向父目录）。
- 任意多个连续的斜杠（即，`'//'` 或 `'///'`）都被视为单个斜杠 `'/'`。
- 任何其他格式的点（例如，`'...'` 或 `'....'`）均被视为有效的文件/目录名称。

返回的 **简化路径** 必须遵循下述格式：

- 始终以斜杠 `'/'` 开头。
- 两个目录名之间必须只有一个斜杠 `'/'` 。
- 最后一个目录名（如果存在）**不能** 以 `'/'` 结尾。
- 此外，路径仅包含从根目录到目标文件或目录的路径上的目录（即，不含 `'.'` 或 `'..'`）。

返回简化后得到的 **规范路径** 。



**分析**：

栈



**代码**：

```java
class Solution {
    public String simplifyPath(String path) {
        // 步骤1：按 / 分割路径为片段（split("/") 会分割出空字符串）
        String[] parts = path.split("/");
        // 步骤2：用栈存储有效目录名（模拟目录进入/回退）
        Deque<String> stack = new ArrayDeque<>();
        
        // 步骤3：遍历所有片段，处理每个部分
        for (String part : parts) {
            // 情况1：空片段（多/分割）或 .（当前目录）→ 忽略
            if (part.isEmpty() || part.equals(".")) {
                continue;
            }
            // 情况2：..（上级目录）→ 栈非空则弹出栈顶（回退）
            else if (part.equals("..")) {
                if (!stack.isEmpty()) {
                    stack.pop();
                }
            }
            // 情况3：正常目录名 → 压入栈（进入该目录）
            else {
                stack.push(part);
            }
        }
        
        // 步骤4：拼接规范路径
        if (stack.isEmpty()) {
            return "/"; // 栈空表示根目录
        }
        StringBuilder sb = new StringBuilder();
        // 栈是后进先出，需从栈底到栈顶拼接（用迭代器或反向遍历）
        for (String dir : stack) {
            sb.insert(0, dir); // 插入到开头，保证顺序正确
            sb.insert(0, "/"); // 每个目录前加 /
        }
        return sb.toString();
    }
}
```
