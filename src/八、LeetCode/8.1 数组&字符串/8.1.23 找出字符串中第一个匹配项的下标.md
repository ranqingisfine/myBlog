# 8.1.23 找出字符串中第一个匹配项的下标

**题目**：

给你两个字符串 `haystack` 和 `needle` ，请你在 `haystack` 字符串中找出 `needle` 字符串的第一个匹配项的下标（下标从 0 开始）。如果 `needle` 不是 `haystack` 的一部分，则返回 `-1` 。

**KMP算法是干嘛的？**：

KMP 算法是一种**高效的字符串匹配算法**，核心解决的问题是：

给定一个「主串」（比如 "ABCDABCDABX"）和一个「模式串」（比如 "ABCDABX"），快速找到模式串在主串中第一次出现的位置（如果不存在返回 -1）。

对比普通的暴力匹配（逐个字符比对，不匹配就回退主串指针），KMP 的优势是**主串指针从不回退**，只靠模式串的「部分匹配表」（也叫 next 数组）让模式串指针智能回退，时间复杂度从暴力的 O (n*m) 降到 O (n+m)（n 是主串长度，m 是模式串长度）。

**KMP的核心思想：利用部分匹配信息，避免重复比对**

先理解一个关键概念：**最长相等前后缀**（这是 next 数组的核心）。

- 前缀：字符串中除了最后一个字符的所有头部子串（比如 "ABCDA" 的前缀是 "A"、"AB"、"ABC"、"ABCD"）。
- 后缀：字符串中除了第一个字符的所有尾部子串（比如 "ABCDA" 的后缀是 "A"、"DA"、"CDA"、"BCDA"）。
- 最长相等前后缀：前缀和后缀中长度最长的相等子串（比如 "ABCDA" 的最长相等前后缀是 "A"，长度 1）。

举个例子：

表格

| 模式串子串 |    最长相等前后缀     | 长度 |
| :--------: | :-------------------: | :--: |
|     A      |    无（单个字符）     |  0   |
|     AB     |          无           |  0   |
|    ABC     |          无           |  0   |
|    ABCD    |          无           |  0   |
|   ABCDA    |  A（前缀）=A（后缀）  |  1   |
|   ABCDAB   | AB（前缀）=AB（后缀） |  2   |
|  ABCDABX   |          无           |  0   |

KMP 的核心就是：当主串和模式串某个位置不匹配时，不是从头重新比模式串，而是根据「最长相等前后缀」的长度，让模式串指针回退到该长度的位置，主串指针继续往后走，从而跳过无效比对。

```java
class Solution {
    // 主方法：实现 strStr 功能（LeetCode 28题），返回needle在haystack中首次出现的索引，不存在则返回-1
    public int strStr(String haystack, String needle) {
        // 边界处理：如果模式串为空，按LeetCode要求返回0
        if (needle == null || needle.length() == 0) {
            return 0;
        }
        // 1. 构建模式串needle的next数组（部分匹配表）
        int[] next = kmpNext(needle);
        // 2. 利用next数组执行KMP匹配，返回匹配结果
        int result = kmpSearch(haystack, needle, next);
        return result;
    }

    /**
     * KMP核心匹配方法
     * @param str1 主串（haystack）
     * @param str2 模式串（needle）
     * @param next 模式串的部分匹配表（next数组）
     * @return 模式串在主串中首次出现的起始索引，未匹配到返回-1
     */
    public static int kmpSearch(String str1, String str2, int[] next) {
        // i：主串指针（永不回退），j：模式串指针（智能回退）
        for (int i = 0, j = 0; i < str1.length(); i++) {
            // 核心逻辑：当主串和模式串当前字符不匹配时，模式串指针j智能回退
            // 这里用while而不是if，是为了处理多次回退的情况（比如j回退后仍不匹配）
            while (j > 0 && str1.charAt(i) != str2.charAt(j)) {
                // j回退到前一个位置的部分匹配值对应的索引（核心！）
                j = next[j - 1];
            }

            // 如果当前字符匹配，模式串指针j后移
            if (str1.charAt(i) == str2.charAt(j)) {
                j++;
            }

            // 终止条件：j等于模式串长度，说明完全匹配
            if (j == str2.length()) {
                // 计算起始索引：主串当前位置 - 模式串长度 + 1
                return i - j + 1;
            }
        }
        // 遍历完主串仍未匹配到，返回-1
        return -1;
    }

    /**
     * 构建模式串的next数组（部分匹配表）
     * @param dest 模式串（needle）
     * @return next数组，next[i]表示模式串前i+1个字符的最长相等前后缀长度
     */
    public static int[] kmpNext(String dest) {
        // 1. 初始化next数组，长度等于模式串长度
        int[] next = new int[dest.length()];
        // 2. 单个字符的最长相等前后缀长度为0（基础规则）
        next[0] = 0;

        // i：遍历模式串的指针（从1开始，因为0位置已初始化）
        // j：表示当前找到的最长相等前后缀的长度（初始为0）
        for (int i = 1, j = 0; i < dest.length(); i++) {
            // 核心逻辑：当当前字符不匹配时，j回退到上一个可能的位置
            // 例如ABACABAB，i=7时，ABA是最大公共前后缀，B和C不匹配，因此j退回找到次长的公共前后缀（A），然后比较B和B匹配，因此i=7位置的j值=2，可以形象的理解为：在字符串找到一段匹配的区域（ABA），当匹配失败时，我们不是完全放弃，而是在这段匹配区域内部，寻找一个更短的“备用”匹配（A），然后从这个备用匹配的末尾，继续尝试与新字符匹配，不断进行上述操作即可
            //即是在匹配失败时，“聪明”的回退到上一个更短的，以验证过的有效长度
            // 同样用while处理多次回退的情况
            while (j > 0 && dest.charAt(i) != dest.charAt(j)) {
                // j回退到前一个位置的部分匹配值（关键！利用已有的匹配信息）
                j = next[j - 1];
            }

            // 如果当前字符匹配，说明最长相等前后缀长度+1
            if (dest.charAt(i) == dest.charAt(j)) {
                j++;
            }

            // 记录当前位置的部分匹配值（即最长相等前后缀长度）
            next[i] = j;
        }
        // 返回构建好的next数组
        return next;
    }
}
```
