# 8.18.3 寻找峰值

leetcode.162

## 1、题目

峰值元素是指其值严格大于左右相邻值的元素。

给你一个整数数组 `nums`，找到峰值元素并返回其索引。数组可能包含多个峰值，在这种情况下，返回 **任何一个峰值** 所在位置即可。

你可以假设 `nums[-1] = nums[n] = -∞` 。

你必须实现时间复杂度为 `O(log n)` 的算法来解决此问题。

**示例 1：**

```
输入：nums = [1,2,3,1]
输出：2
解释：3 是峰值元素，你的函数应该返回其索引 2。
```



## 2、分析

题目：nums[-1] = nums[n] = -∞，两边都是负无穷大，则中间肯定有一个峰值，因此可以用二分法来找这个峰值。

- 如果nums[mid]<nums[mid+1]，则右边更大，峰值在右边，而mid肯定不是峰值，可以放心left=mid+1让掉它
- 如果nums[mid]>nums[mid+1]，则左边更大，峰值在左边，而mid可能是峰值，必须right=mid保留它

**`left <= right` 适合 “精确查找某个值”**

**`left < right` 适合 “找极值、找边界、找满足条件的位置”**

LeetCode 162 是**找峰值（极值）**，不是找某个固定值，所以天生适合 `left < right`。

### 用了 `right = mid`，就不能搭配 `left <= right`

因为会出现：

```
left == right == mid
```

然后再次进入循环

再执行 `right = mid`

区间永远不变 → **死循环**

例如：

```
[1, 3, 2]
最后一步：left = 1, right = 1
进入 while(left <= right) → 条件成立mid = 1比较 nums [1] 和 nums [2]执行 right = mid → right 还是 1无限循环
这就是为什么 162 不能用 left <= right。
```

## 3、代码

```java
class Solution {
    public int findPeakElement(int[] nums) {
        int left = 0;
        int right = nums.length - 1;
        
        while (left < right) {
            int mid = left + (right - left) / 2;
            // 右边更大，去右边
            if (nums[mid] < nums[mid + 1]) {
                left = mid + 1;
            } else {
                // 左边更大，去左边
                right = mid;
            }
        }
        return left;
    }
}
```
