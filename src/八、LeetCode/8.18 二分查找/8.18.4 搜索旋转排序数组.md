# 8.18.4 搜索旋转排序数组

leetcode.33

## 1、题目

整数数组 `nums` 按升序排列，数组中的值 **互不相同** 。

在传递给函数之前，`nums` 在预先未知的某个下标 `k`（`0 <= k < nums.length`）上进行了 **向左旋转**，使数组变为 `[nums[k], nums[k+1], ..., nums[n-1], nums[0], nums[1], ..., nums[k-1]]`（下标 **从 0 开始** 计数）。例如， `[0,1,2,4,5,6,7]` 下标 `3` 上向左旋转后可能变为 `[4,5,6,7,0,1,2]` 。

给你 **旋转后** 的数组 `nums` 和一个整数 `target` ，如果 `nums` 中存在这个目标值 `target` ，则返回它的下标，否则返回 `-1` 。

你必须设计一个时间复杂度为 `O(log n)` 的算法解决此问题。

**示例 1：**

```
输入：nums = [4,5,6,7,0,1,2], target = 0
输出：4
```

题目的核心问题是：将数组根据某个位置旋转后，找数组中target的下标，如果没有就返回-1，不要理解错了

## 2、分析

核心一句话：

**旋转后数组仍有一半是有序的，每次二分判断哪一半有序，再看 target 在不在有序那一半。**

核心思路：

```
每次取 `mid`，**必然有一半是完全有序的**：
- 要么 **左半边有序**：`nums[left] ≤ nums[mid]`
- 要么 **右半边有序**
然后只做一件事：
**看 target 是否落在【有序的那一半区间内】**
- 在 → 只搜这一半
- 不在 → 去另一半
```

### ① 左半边有序（`nums[left] ≤ nums[mid]`）

看 target 是否在 `[nums[left], nums[mid]]` 之间：

- **在**：`right = mid - 1`
- **不在**：`left = mid + 1`

### ② 右半边有序（else）

看 target 是否在 `[nums[mid], nums[right]]` 之间：

- **在**：`left = mid + 1`
- **不在**：`right = mid - 1`

---

**为什么不是** **`left < right`****？**

- `left < right` 常用于 **收敛型二分**（比如找峰值）
- 循环结束时 `left == right` → 区间只剩一个元素 → 返回 `left`

而这里我们要做的是：

1. 可能 **正好找到目标值** → `return mid`
2. 可能 **整个区间没找到** → 返回 -1

所以要保证 **最后一个元素也被检查**，必须用：left <= right

- `while (left <= right)` → **区间闭合** **`[left,right]`**，保证每个元素都被考虑
- `while (left < right)` → **区间半开** **`[left,right)`**，适合收敛型问题，通常返回 `left` 或 `right`
- 在 **旋转数组搜索**中，我们需要遍历整个区间找到目标 → 用 `<=` 最稳妥



## 3、代码

```java
class Solution {
    public int search(int[] nums, int target) {
        int left = 0;
        int right = nums.length - 1;

        while (left <= right) {
            int mid = left + (right - left) / 2;

            if (nums[mid] == target) return mid;

            // 左半边有序
            if (nums[left] <= nums[mid]) {//注意，这是小于等于，等于是保证单元素仍然被判为有序
                // target 在左半边
                if (target >= nums[left] && target < nums[mid]) {
                    right = mid - 1;//因为程序走到这里来nums[mid]!=target的，所以right和left的值都要排查mid
                } else {
                    left = mid + 1;
                }
            } 
            // 右半边有序
            else {
                // target 在右半边
                if (target > nums[mid] && target <= nums[right]) {
                    left = mid + 1;
                } else {
                    right = mid - 1;
                }
            }
        }
        return -1;
    }
}
```
